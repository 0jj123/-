☁️ 雲端服務

1.3 — 網際網路架構

網際網路由多個 ISP（Internet Service Provider） 互相連接組成，核心稱為 Internet Core。組織（如 MIT、Amazon）透過 ISP 接入網際網路，家庭也可透過小型網路連接。

1.4 — Netflix & AWS

Netflix 不自建伺服器，而是租用 Amazon Web Services (AWS) 的伺服器農場（Server Farm）。AWS 提供多個區域伺服器農場，並以冗餘（Redundancy）設計確保容錯能力（Fail-Over）。

1.6 — 伺服器虛擬化

虛擬化讓一台實體伺服器可同時執行多個 虛擬機器（VM），每個 VM 都像獨立伺服器。優點：


• 可在數秒內透過自助工具建立新 VM

• VM 可輕鬆搬移到其他實體伺服器（整個 VM 是一個檔案）

• 暫時新增更多 VM 來因應流量高峰

• 提升組織的 Agility（敏捷性）——快速調整的能力

1.7 — 雲端服務模型比較

Server Owner（伺服器擁有者）Application Owner（應用擁有者）
Corporate Data Center（自建機房）用戶組織用戶組織
IaaS（基礎設施即服務）IaaS CSP用戶組織
SaaS（軟體即服務）SaaS CSPSaaS CSP

傳統模式：組織自行購買並維護伺服器和軟體（as a product）
雲端模式：不買斷，按使用量付費（as a service），如同水電費

🌐 網路基礎概念

1.1 — 網路通訊基本模型

任何連接到網路的電腦都是 Host（主機），包括伺服器、PC、手機、智慧手錶等。隨著萬物互聯，這個概念延伸為 IoT（Internet of Things）。

基本通訊模型：Client Host（瀏覽器）→ 網際網路 → Server Host（Web 伺服器程式）

1.2 — 點分十進位記法（Dotted Decimal Notation）

IP 位址為 32 bits，分成四個 8-bit 段，每段轉換為十進位後以「.」分隔。


例：00000001.00000010.00000011.00000100 → 1.2.3.4

1.11–1.14 — 訊息、分段與多工

HTTP Request-Response Cycle：瀏覽器送出 HTTP Request，伺服器回應 HTTP Response。


Fragmentation（分段）：長訊息切成小段（Segment），加上標頭（Header）後形成 Frame 或 Packet。好處：出錯時只需重傳出錯的那個 Frame，不需重傳整個訊息。


Multiplexing（多工）：多個主機的 Frame 混合共用同一條傳輸線路，降低每位使用者的成本。

1.15 — EUI-48 位址（MAC 位址）

EUI-48 (Extended Unique Identifier) = 舊稱 MAC Address，長度 48 bits，以 6 對十六進位數字表示，用破折號分隔。

範例：A1-B2-CC-92-FF-00

使用於：Ethernet 802.3、Wi-Fi 802.11、Bluetooth

1.22–1.23 — 跨網路連線與路由器

直接連接兩個不同的單一網路面臨三大問題：標準不同、位址重疊、連線媒介不一致。

解決方案：Router（路由器），舊稱 Gateway（閘道器）。路由器在不同網路間扮演目的主機或來源主機，將封包從一個 Frame 取出後放入另一個 Frame 繼續傳送。

小型家庭網路元件

Wireless Access Router（無線存取路由器） 包含：路由器 + 乙太網路交換器 + 無線存取點 + DHCP 伺服器 + NAT 功能


DHCP：動態分配 IP 位址。ISP 的 DHCP 給家庭一個對外 IP；家中路由器的 DHCP 再給各設備不同的內部 IP（如 192.168.0.x）。


NAT（網路位址轉換）：將家庭內部 IP（如 192.168.0.6）轉換為對外公共 IP（如 60.47.243.112），讓多台設備共用一個公共 IP 上網。

1.31 — DNS

Domain Name System：將網域名稱（如 voyager.shidler.hawaii.edu）轉換為 IP 位址（如 128.171.17.13）。步驟：


① 來源主機想傳送封包給某網域名稱

② 向 DNS 伺服器查詢該名稱的 IP 位址

③ DNS 伺服器回傳對應 IP

④ 來源主機再以此 IP 傳送封包

🔢 標準分層架構

5
Application Layer（應用層） 規範兩端應用程式的通訊格式（如 HTTP、DNS）
4
Transport Layer（傳輸層） 分段（Fragmentation）及其他功能（如 TCP）
3
Internet Layer（網際網路層） 封包跨網路傳輸，路由器運作，主標準：IP（Layer 3 / L3）
2
Data Link Layer（資料連結層） Frame 在單一網路傳輸，交換器/AP 運作，標準：Ethernet 802.3、Wi-Fi 802.11、PPP（Layer 2 / L2）
1
Physical Layer（實體層） 傳輸媒介、接頭、訊號（Layer 1 / L1）

3.1

The Target Breach — 2013 年零售業史上最大資料洩漏

事件背景

  • 2013 年聖誕購物季，Target 百貨遭受大規模入侵
  • 使用 BlackPOS 惡意程式，從 POS 終端機「刮取」刷卡資料
  • 4,000 萬筆信用/借記卡交易資料外洩；另一起攻擊另取得 7,000 萬筆顧客個人資訊
  • 後果：業績下滑、股價崩跌、資訊長被開除、CEO 被迫下台

攻擊流程（完整鏈條）：

1
Crimeware Shop 販售 BlackPOS 給攻擊者
2
攻擊者對 Fazio Mechanical Services（Target 供應商）發動 Spear Phishing 電郵攻擊，取得存取憑證
3
透過 Vendor Server 進入 Target 內部網路
4
Malware Download Server 將 BlackPOS 推送到所有 POS Terminals
5
POS 機刷卡資料即時送往 Holding Server
6
Extrusion Server 將資料傳送到外部 Landing Servers
7
資料流向 Card Shops 後被 Card Counterfeiter（偽造者）製作偽卡
關鍵補充：整個過程中還有來自外部的持續 Command & Control (C&C) 通訊，但圖中未顯示。
3.2

Malware and Vulnerabilities

Malware（惡意軟體）

泛指所有惡意軟體的統稱，包含病毒、蠕蟲、木馬、勒索軟體等。

Vulnerabilities（安全漏洞）

  • 特定程式中的安全缺陷
  • 廠商發布 Patch（修補程式）來修補
  • 使用者若未即時安裝，仍處於風險中
  • Zero-day Attack：在 patch 釋出前即發動的攻擊
3.3

Viruses and Worms

🦠 Virus（病毒）

需要附著在其他程式上才能傳播的惡意程式碼片段；宿主程式被執行時病毒才會啟動。

🐛 Worm（蠕蟲）

獨立執行的程式，不需附著於其他程式。可自行在網路上傳播。

傳播途徑說明
電子郵件附件使用者開啟附件即感染（需人為操作，速度較慢）
瀏覽網站即使是合法網站也可能含有惡意腳本
社交網站釣魚連結、惡意貼文
USB、P2P 分享實體媒介或檔案共享服務
Directly Propagating Worm直接跳到有漏洞的目標主機，無需使用者操作，傳播速度極快。2003 年 SQL Slammer 在 10 分鐘內感染 75,000 台主機。
3.4–3.5

其他惡意程式類型 & Payloads

Mobile Code（行動程式碼）

  • 網頁中的腳本（Script），隨網頁下載到瀏覽器
  • 正常情況下無害（benign 良性的）
  • 若瀏覽器有漏洞則可能造成損害

Trojan Horse（木馬程式）

  • 偽裝成系統檔案的惡意程式
  • Spyware Trojan：竊取敏感資料後回傳給攻擊者

Downloader（下載器）

先感染電腦後，再下載更大的惡意程式。

Spam（垃圾郵件）

未經要求的商業電子郵件（Unsolicited Commercial E-Mail）。

Payload（酬載）

  • 病毒/蠕蟲傳播後執行的破壞動作
  • 惡意 Payload：抹除硬碟、將使用者導向惡意網站等
  • 即使沒有 Payload，惡意程式仍可造成系統效能下降或當機
3.6

Attacks on Human Judgment — 社交工程攻擊

Social Engineering
欺騙受害者做出違反個人或組織安全利益行為的攻擊手法。
例：誘使開啟附件、造訪惡意網站。
Phishing
以偽造成真實外觀的電郵或網站誘騙使用者輸入帳號密碼等敏感資訊。
Spear Phishing
針對特定個人的精準魚叉式網路釣魚，Target Breach 即使用此手法攻擊 Fazio。
Credit Card Theft
竊取信用卡號後進行未授權交易。
Identity Theft
收集足夠資料冒充受害者進行大型金融交易；危害遠大於信用卡盜用，且恢復信用評分可能耗費大量時間。
3.7–3.10

Hacking / DDoS / APT

Hacking
Intentionally 使用電腦資源但未獲授權，或超出授權範圍。核心關鍵字：Authorization（授權）。
Exploit
實際的入侵行為；執行入侵的工具也稱為 Exploit。入侵後可手動操作，或留下木馬持續自動竊取。

DDoS — 分散式阻斷服務攻擊

  • DoS：讓電腦或整個網路對合法使用者無法使用（影響 Availability）
  • DDoS：透過大量 Bots（殭屍電腦）同時發動攻擊
  • 受控制的電腦集合稱為 Botnet（殭屍網路）
  • Botmaster 透過 Command & Control (C&C) Server 下令
  • SYN Flood：連續發送大量 SYN 封包耗盡伺服器資源，使合法連線無法建立

APT — 進階持續性威脅（3.10）

  • 針對單一組織的長期攻擊（可持續數月至數年）
  • 使用極為先進的技術；通常以 Spear Phishing 作為入口
  • 成本高昂 → 通常由國家政府執行，但資金充足的犯罪組織也可做到
3.11

Types of Attackers — 攻擊者分類

Hackers（駭客）

  • Old-school：好奇心驅動，追求聲望與權力
  • 現代：多為職業犯罪駭客，以金錢為目的
  • 有完善的線上犯罪基礎設施：購買惡意程式、租用專業技術、信用卡銷贓

Malware Attackers

  • 撰寫惡意程式：不違法
  • 對公司釋放惡意程式：違法

Insiders（內部人員）

  • 現任員工（復仇或竊盜）或前員工
  • 極為危險：已有存取權、熟悉系統、知道如何規避偵測、被組織信任
  • IT/安全人員最危險；離職前需立即終止所有存取
  • 有存取權限的承包商也視為內部人員

Cyberterrorists

  • 恐怖份子或國家發動的網路攻擊
  • 技術複雜度高，目標是大規模破壞而非個別犯罪
3.12

Symmetric Key Encryption — 對稱金鑰加密

Cryptography（密碼學）：利用數學方法保護資訊的科學。
Symmetric Key
兩端共享同一把金鑰進行加密與解密。最主流的對稱加密演算法為 AES（Advanced Encryption Standard）；金鑰長度至少需 100 bits，標準使用 128 bits。
步驟發送端 (Party A)網路傳輸接收端 (Party B)
加密明文 + 對稱金鑰 + Cipher → 密文密文（竊聽者無法解讀）密文 + 相同對稱金鑰 + Cipher → 明文

Public-Key Cryptography（非對稱）— 加密機密性

  • 每方各有一對金鑰：Public Key（公開）+ Private Key（私密）
  • 加密保密性：用接收者 Public Key 加密 → 只有接收者用 Private Key 解密
  • 常見演算法：RSA

Public-Key Cryptography — 身份驗證

  • 任一把金鑰都能加密，另一把解密
  • 用發送者 Private Key 加密 → 接收者用 Public Key 解密 → 確認身份
3.13

Electronic Signature Authentication — 數位簽章

Authentication（認證）

驗證傳送者身份的過程。

Message Integrity（訊息完整性）

接收端能夠偵測訊息在傳輸過程中是否被篡改。

Electronic Signature（ES） 同時提供 Authentication 和 Message Integrity。
3.14

VPN / SSL/TLS

SSL/TLS VPN
在不可信任的網路（Internet、無線網路）中建立加密保護的傳輸通道。
SSL = Secure Socket Layer（Netscape 開發）
TLS = Transport Layer Security（IETF 標準化），現為主流
已廣泛支援於各大瀏覽器。
3.15–3.19

Authentication — 身份驗證方法

Credential
身份證明文件（e.g., 密碼、卡片、生物特徵）。由 Supplicant（申請者）送交 Verifier（驗證者）。
方法優點缺點/限制
Reusable Password
可重複使用密碼		 熟悉、免費（內建於 OS） 易被 Dictionary Attack 破解；複雜密碼常被寫下來
Access Card
門禁卡		 可控制實體與電腦存取；Proximity Card 不需接觸 可能遺失或被複製
Fingerprint Recognition
指紋辨識		 便宜、低風險應用已足夠 精準度低、可被欺騙
Iris Recognition
虹膜辨識		 高精準度、難以欺騙 昂貴
Facial Recognition
臉部辨識		 可遠端/隱性辨識 爭議性大（可在當事人不知情下進行）
Digital Certificate
數位憑證		 高安全性；Challenge-Response 機制 需 CA 基礎設施
Two-Factor Authentication
雙因子驗證		 需兩種憑證（e.g., 信用卡 + PIN）；強化安全性 攻擊者控制電腦或能攔截通訊時仍可失效

Password Security

  • Dictionary Attack：使用常見密碼字典，幾秒至數分鐘破解大多數密碼
  • Brute Force Attack：嘗試所有字符組合；長且複雜的密碼只能靠此法破解，通常不可行
  • 強密碼原則：最少 8–12 字符、混合大小寫/數字/特殊符號（e.g., r8pWm#4D*&2B）

Digital Certificate 驗證流程（3.18）

1
Verifier 傳送 Challenge Message 給 Supplicant
2
Supplicant 用自己的 Private Key 加密 Challenge → 產生 Response Message
3
Supplicant 回傳 Response
4
Verifier 向 CA（Certificate Authority） 取得 True Party 的 Digital Certificate（含 Public Key）
5
Verifier 用 True Party 的 Public Key 解密 Response
6
若解密成功還原原始 Challenge → 確認 Supplicant 持有 True Party 的 Private Key → 身份驗證通過
3.20–3.25

Firewalls — 防火牆

General Firewall Operation（3.20）

  • 放置於內部網路與外部之間的 Border Firewall
  • Ingress Filtering：過濾入站封包
  • Egress Filtering：過濾出站封包
  • 確認的攻擊封包 → Denied and Logged（拒絕並記錄）
  • 無法確定的可疑封包處理方式影響安全與可用性的平衡

SPI — Stateful Packet Inspection（3.21–3.23）

利用連線狀態判斷封包合法性：

Connection-Opening Attempt（~1% 封包）

高風險 → 必須驗證；透過 ACL（Access Control List）逐條比對決定允許或拒絕

Ongoing Communication（~99% 封包）

若屬於已許可的連線 → Accept
若非已許可的連線 → Drop

ACL 規則匹配原則：由上至下逐條比對，第一條符合的規則即執行；最後一條通常為「拒絕所有」(Deny All)。

ACL 範例（Approved Connections Table 另外記錄已許可連線的 IP+Port）：

RuleDestination IPService (Port)Action
1ALL25 (SMTP)✅ Allow connection
210.47.122.7980 (HTTP)✅ Allow connection
3ALLALL🚫 Do not allow connection

NGFW — Next-Generation Firewall（3.25）

  • SPI 的限制：只能檢查 Socket 資料（IP + Port），無法識別 Port 80 上跑的是哪種應用
  • NGFW 使用 Deep Inspection：檢查所有網路/傳輸層欄位，還能檢查應用層內容（需重組多個 segment）
  • Application Awareness：能識別產生流量的應用，甚至能辨識惡意應用
  • IDS（Intrusion Detection System）：偵測可疑流量並記錄/通知；可能產生大量誤報（false alarms）
  • IPS（Intrusion Prevention System）：IDS + 自動丟棄高信心度攻擊封包
  • Reputation Management：使用外部黑白名單資料庫
  • Wire-Speed Operation：NGFW 需使用專用硬體（不是一般電腦），才能在高速傳輸線路上及時處理大量封包
3.26

Antivirus Protection — 防毒保護

🔥 Firewall

處理封包層級的過濾

🛡️ Antivirus

處理檔案層級的掃描；不限於病毒，針對所有形式的惡意程式

Signature Detection（特徵碼偵測）

  • 比對已知惡意程式的位元組特徵碼
  • 問題1：惡意程式數量已太多，無法涵蓋所有特徵碼
  • 問題2：許多惡意程式會突變（mutate）改變特徵碼

Behavioral Detection（行為偵測）

  • 分析程式嘗試執行的行為（e.g., 格式化硬碟）
  • 可在沙箱（Sandbox）中安全執行可疑程式來研究其行為
  • 若判定有害則刪除

5.1 Simple Switched Ethernet Network 架構
🏢 Core Switch（核心交換器）
  • 負責 交換器對交換器 的連接
  • 使用 光纖（Optical Fiber） 作為 Trunk Lines
  • 位於網路階層最上層（A、B、C）
🖥️ Workgroup Switch（工作群組交換器）
  • 負責 Host 連接到網路
  • 使用 UTP 銅線 作為 Access Lines
  • 連接終端設備（D、E、F）
⚠️ 重要觀念
LAN (Local Area Network) = 建置在客戶場所（customer premises）內的網路。Workgroup Switch 連接 Hosts；Core Switch 連接 Switches。
📐 5.2 — Workgroup Switch 規格

標準 1U 機架式，寬 19 吋 / 48 cm，高 1U = 1.75 吋，可安裝於標準設備機架（Rack）。典型規格：48 個 UTP 埠 + 若干光纖上行埠。

5.6 Binary vs. Digital Signaling
⬛ Binary Transmission（二進位傳輸）
  • 只有 2 種狀態（0 / 1）
  • 每個 Clock Cycle 傳送 1 bit
  • 方波訊號，高低電位切換
🔢 Digital Transmission（數位傳輸）
  • 有 多種狀態（2, 4, 8, 16…）
  • 每個 Clock Cycle 傳送 >1 bit
  • 例：4 狀態 → 每週期送 2 bits
5.7 — Error Resistance（抗雜訊能力）

電壓範圍分區設計讓訊號具有容錯能力。例如：

  • +3 ~ +15V → 代表 0，-3 ~ -15V → 代表 1
  • 即使訊號從 +12V 衰減到 +6V（衰減 50%），仍正確讀作 0
  • Attenuation（衰減）：訊號強度隨距離遞減
5.8 – 5.11 UTP (Unshielded Twisted Pair) 銅線
📌 UTP 結構
  • 4 對絞合銅線（8 條導線）
  • 連接器：RJ-45 Connector 插入 RJ-45 Jack
  • 每根絞線對彼此絞合以抵抗 Crosstalk
5.10 — Serial vs. Parallel Transmission
  • Serial：單線，每 Clock Cycle 傳 1 bit
  • Parallel：多線對同時傳輸，每對 1 bit → 4 對 = 4 bits/cycle
  • ⚠️ Parallel 不限於 4 對，可用更多對
Ethernet 標準 速度 UTP Category 最大距離

數字越大品質越好，雜訊抑制能力越強。Cat 6A 能在 10Gbps 下達到完整 100m。
5.12 Crosstalk Interference（串音干擾）
🔊 EMI / Crosstalk
  • 每對線向環境輻射，並接收外部輻射
  • 相鄰線對之間互相干擾 → Crosstalk
  • 絞合（Twist）可消除大部分 Crosstalk
⚡ Terminal Crosstalk（端點串音）
  • 線末端需拆開插入 RJ-45 → 無絞合保護
  • 此處 Crosstalk 最嚴重
  • Category 等級越高，端點抑制越好
5.13 – 5.16 光纖傳輸（Optical Fiber）
特性UTP光纖
介質銅線玻璃
訊號電光
LAN 最大距離通常 100 m通常 200~500 m
速度相近相近
成本較低較高
🔬 光纖傳輸原理
  • 光線打到 Cladding（覆層）時產生 完全內部反射（Perfect Internal Reflection）
  • 無訊號損耗（只要不超過最大距離）
  • 光源波長約 850 nm（短波長，用於 LAN）
  • 較長波長 → 傳輸距離更遠，但成本更高
🟠 Multimode Fiber（多模光纖）
  • 核心直徑 50 微米
  • 有 Modal Dispersion（模態色散）
  • 不同角度光線到達時間不同 → 可能重疊出錯
  • ✅ LAN 常用，成本較低
🔵 Single-mode Fiber（單模光纖）
  • 核心直徑 8~9 微米（極細）
  • 無 Modal Dispersion
  • 只允許一種模態（直線穿過）
  • ⚠️ 成本較高，多用於長距離 WAN

5.22 Single Point of Failure（單點故障）
❌ 問題

階層式網路中，若一台 Core Switch 故障，其下所有 Hosts 與其他部分的通訊完全中斷。每台交換器都是潛在的 Single Point of Failure。

5.23 Backup Link & RSTP（快速生成樹協定）
🔁 解決方案：備援鏈路 + RSTP
  • 在兩台 Switch 間加裝 Backup Link 提供備援
  • 但 Ethernet 不允許迴圈（Loop）存在
  • RSTP（Rapid Spanning Tree Protocol） = 802.1w
  • 正常運作時：RSTP 停用備援鏈路（防止迴圈）
  • 主鏈路故障時：RSTP 自動啟用備援鏈路
  • Ethernet 需維持嚴格階層（strict hierarchy）
❗ 重要
Loops are not allowed in Ethernet. RSTP 解決備援與無迴圈的矛盾。
5.24 Advanced Ethernet Capabilities
🎯 Priority（優先權）
  • 支援最多 8 個優先權等級
  • 透過 Tag Fields 攜帶優先權資訊（802.1p QoS）
  • 適用於語音、影像等對延遲敏感的流量
🛠️ Manageability（可管理性）
  • Managed Switch 可透過 SNMP 管理
  • 雖然成本較高，但管理成本大幅降低
  • 未來：Software-Defined Networking（SDN）可能革新管理方式
⚡ PoE（Power over Ethernet）
  • 交換器透過 UTP 線材供電給設備
  • 同一條線傳輸資料 + 供電
  • 比單獨佈電線更省錢
⚡⚡ PoE Plus
  • 最多提供 25 Watts 電力
  • 足夠：VoIP 電話、監控攝影機、AP
  • 不足：桌機或筆電
  • 無需額外佈線節省成本

5.25 802.1X Port-Based Access Control
⚠️ 問題
任何人只要進入建築物，就能將電腦插入交換器或牆壁的 RJ-45，繞過防火牆直接存取網路。
✅ 解決方案：802.1X Port-Based Access Control
  • 由 802.1 WG（非 802.3）制定
  • 在 Switch Port 層級進行存取控制
角色說明
Supplicant（Peer）試圖連接的設備，必須先通過驗證才能使用 Port
Authenticator（Switch）Workgroup Switch，控制 Controlled Port（未授權/已授權）
Central Auth Server儲存驗證資料庫，執行實際驗證工作
🏆 Central Authentication Server 的優勢
  • Switch 成本↓：繁重驗證工作由 Auth Server 負責
  • 一致性↑：攻擊者無法找到設定錯誤的 Switch
  • 管理成本↓：只需維護一個驗證資料庫
  • 變更快速：人員離職/入職/權限異動即時生效
5.26 ARP Poisoning — Man-in-the-Middle Attack
📚 ARP (Address Resolution Protocol) 基礎
  • 乙太網路需要知道目標的 MAC 位址才能傳送 Frame
  • IP 層只知道 IP 位址，不知道 MAC
  • ARP 負責將 IP → MAC 的解析
  • 每台主機維護一個 ARP Cache（IP ↔ MAC 對應表）
☠️ ARP Poisoning 攻擊流程
Step 1 攻擊者廣播假的 ARP 回覆：「IP 1.2.3.4 的 MAC 是 FF-FF-FF-DD-DD-DD（攻擊者）」
Step 2 受害者 ARP Cache 被毒化（Poisoned）：1.2.3.4 → FF-FF-FF-DD-DD-DD
Step 3 受害者傳送封包給「預設路由器」，卻送到攻擊者
Step 4 攻擊者攔截所有流量 → Man-in-the-Middle
🛡️ 防禦：DAI (Dynamic ARP Inspection)
  • Managed Switch 的安全功能
  • 檢查 LAN 上的 ARP 封包
  • 與 DHCP Snooping Table 比對驗證
  • 阻擋不合法的 ARP 回覆
🖥️ ARP 常用指令（Windows）
  • arp -a — 顯示 ARP Cache
  • arp -d * — 清除 ARP Cache
  • arp -s [IP] [MAC] — 手動設定靜態 ARP

Workgroup Switch 與 Core Switch 的差異？
Workgroup Switch 連接 Hosts（Access Lines, UTP）；Core Switch 連接 Switches（Trunk Lines, 光纖）。
點擊翻面
Binary vs. Digital Transmission 的差異？
Binary = 2 狀態，每週期 1 bit；Digital = 多狀態（4/8/16…），每週期 >1 bit。
點擊翻面
UTP 為何要「絞合」？
絞合使相鄰線對的電磁干擾互相抵消，有效減少 Crosstalk Interference。
點擊翻面
Terminal Crosstalk 發生在哪裡？為什麼？
發生在線材末端插入 RJ-45 的地方，因為線需要拆開（untwisted）才能插入，失去絞合保護。
點擊翻面
10GBASE-T 用 Cat 6 vs Cat 6A 的距離差異？
Cat 6 → 55 m；Cat 6A → 100 m。同樣 10 Gbps，但 Cat 6A 品質更好能達到更遠距離。
點擊翻面
Multimode vs Single-mode 光纖？
Multimode：50μm，有 Modal Dispersion，LAN 常用；Single-mode：8-9μm，無 Modal Dispersion，成本更高。
點擊翻面
什麼是 Modal Dispersion？
多模光纖中不同角度（模態）的光線行進距離不同，到達時間不同，若不同時鐘週期的光重疊會造成錯誤。
點擊翻面
Switch 是「再生」還是「放大」訊號？差別？
Switch 是「再生（Regenerate）」—— 讀取訊號後以全強度重新發送，而不是放大（含雜訊的）舊訊號。
點擊翻面
802.3 Frame 的 FCS 欄位功能？
Frame Check Sequence，4 Octets，接收端用來偵測 Frame 在傳輸過程中是否發生錯誤。
點擊翻面
IEEE 802.1X 的三個角色？
①Supplicant（申請連接的設備）②Authenticator（Switch，控制 Port）③Central Auth Server（驗證資料庫）
點擊翻面
RSTP 的作用是什麼？
防止 Ethernet 迴圈（Loop）：正常時停用備援鏈路，主鏈路故障時自動啟用備援鏈路，維持嚴格階層拓撲。
點擊翻面
ARP Poisoning 如何實現中間人攻擊？
攻擊者送假 ARP 回覆，讓受害者 ARP Cache 以為路由器 IP 對應攻擊者的 MAC，使流量轉送給攻擊者。
點擊翻面
DAI 如何防禦 ARP Poisoning？
Dynamic ARP Inspection 比對 ARP 封包中的 IP-MAC 對應是否與 DHCP Snooping Table 一致，不合法則丟棄。
點擊翻面
PoE Plus 最多提供多少電力？適用哪些設備？
最多 25 Watts，適用 VoIP 電話、監控攝影機、無線 AP；不足以供給桌機或筆電。
點擊翻面
Link Aggregation 的作用？
將多條並行實體鏈路聚合為一條邏輯鏈路，提升頻寬並增加備援能力（例：2 × 1Gbps = 2Gbps trunk）。
點擊翻面
802.1X 由哪個 Working Group 制定？為什麼？
802.1 WG（非 802.3 WG）制定，因為 802.1X 是跨媒介的安全標準，802.1 WG 負責通用與安全標準。

A2 Octets，標示 Frame 長度
B8 Octets，LLC 子標頭
C4 Octets，錯誤偵測

B允許 Ethernet 中存在迴圈

Q9. ARP Poisoning 攻擊的核心手法是？
A暴力破解路由器密碼
B發送假 ARP 回覆，污染受害者的 ARP Cache

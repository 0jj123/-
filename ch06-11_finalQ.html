無線區域網路 I — Wi-Fi 基礎

802.11 標準 · 電波傳播 · 頻段頻寬 · 展頻 · WLAN 運作 · 傳輸標準 · 網狀網路
本章建立 Wi-Fi 的物理與運作基礎。核心主線:無線電波先天不可靠(衰減、干擾、多路徑),所以 802.11 用各種技術(展頻、CSMA/CA、MIMO)在「共享又混亂」的無線媒介上盡力可靠通訊。每一個「問題」都是下一個「技術」的動機。

p.3–7802.11 標準與 Wi-Fi 聯盟

有線乙太網路 802.3 與無線 802.11 是 IEEE 802 委員會底下的兄弟標準,都只負責 OSI 的第 1 層(實體)與第 2 層(資料鏈結)。差別只在傳輸媒介:銅線 vs 無線電波。第 3 層以上(IP/TCP)由別的標準負責,與它們無關。

🔑 IEEE 與 Wi-Fi 聯盟的分工(必考)

IEEE 制定 802.11「技術標準」;Wi-Fi 聯盟從標準中挑出功能子集(profile)做「互通性認證」,通過才能貼 Wi-Fi 標誌。所以「Wi-Fi」嚴格說是認證商標,不是標準名稱。

Wi-Fi 聯盟偶爾也自己推標準,出過兩個「資安惡夢」:WPA(WEP 被破後的過渡方案,TKIP 已不安全)與 WPS(8 位數 PIN 被拆成 4+3 段驗證,暴力破解只需約 11,000 次,建議關閉)。

混合式網路(Hybrid Network)

真實 Wi-Fi 幾乎都是「最後一段無線」:筆電 →(無線電)→ AP →(UTP 網路線)→ 乙太交換器 → 伺服器/Internet。AP 扮演 Layer 2 橋接器,把 802.11 訊框轉成 802.3 訊框。無線只是有線網路的延伸,不是取代。

p.10–21電波傳播與五大問題

電磁波三要素:振幅(訊號強度)、波長 λ、頻率 f,關係 c = f × λ。光纖習慣講波長,無線電習慣講頻率。典型資料 LAN 落在 500 MHz–10 GHz(Wi-Fi 的 2.4/5 GHz 在此)。

🔑 反平方定律計算

強度 ∝ 1/r²,公式 S₂ = S₁ × (r₁/r₂)²(新距離放分母)。例:10 m 處 9 mW,30 m 處 = 9 × (10/30)² = 9 × 1/9 = 1 mW。距離 ×3 → 功率 ÷9。

p.22–36頻段、頻道與頻寬

三層架構:頻譜(0 Hz→∞ 所有頻率)→ 服務頻段(劃給特定用途的區塊,多數需執照)→ 頻道(頻段內細分,實際開台的單位)。不同頻道訊號互不干擾。

🔑 頻寬 ∝ 速度

頻寬 = 最高頻率 − 最低頻率(單位 Hz)。頻寬加倍 → 傳輸速度上限加倍(香農定理 C = B·log₂(1+SNR) 中 C ∝ B)。這就是 Wi-Fi 拼命搶寬頻道的根本原因。「broadband 寬頻」工程定義是頻道頻寬寬,不是單純「快」。

p.43–47展頻傳輸與 OFDM

🔑 展頻三大要點(必考)

① 在 2.4/5 GHz 法律強制使用展頻;② 能減輕傳播問題,尤其多路徑;③ 在 WLAN 中不是用來做安全的(參數公開,毫無保密效果——安全靠 WPA2/3 加密)。

展頻 = 故意用遠超必要的頻寬傳訊號,把能量攤薄。某個窄頻率被干擾或抵消,其他頻率成分還活著 → 抗多路徑、抗 EMI。「浪費頻寬但為避免傳播問題是必要的」。注意:展頻買到的是可靠性,不是速度。

OFDM(正交分頻多工):把寬頻道切成幾十~幾百條窄子載波,一個訊框拆散並行傳送,資料以冗餘方式分佈。某些子載波被多路徑抵消,其餘仍能重建資料。是 802.11a/g/n/ac/ax、4G、5G 的現役主力技術。

p.49–68802.11 WLAN 運作

封包與訊框之旅(必考觀念)

筆電送資料給有線伺服器:封包(Layer 3)裝進 802.11 訊框無線送到 AP → AP 拆掉 802.11 訊框、取出封包 → 重新裝進 802.3 訊框走有線送到伺服器。

核心口訣封包走全程(端到端不變),訊框只走一段(per-hop)。一個封包,N 段鏈路 = N 個不同訊框。AP 是唯一的雙語翻譯官;交換器不拆訊框,路由器才會。

有線骨幹在 802.11 標準術語叫 DS(分送系統)。架構層級:STA(站台)→ BSS(一台 AP + 它的客戶)→ DS(串接多 AP 的有線骨幹)→ ESS(多個 BSS + DS,共用同一 SSID)。SSID = 網路名稱;同 ESS 同 SSID 才能漫遊(hand off)。

媒介存取:CSMA/CA + ACK

一個 BSS 共用一個頻道,同時只能一人講話,否則碰撞。無線發射時聽不到別人,無法偵測碰撞,所以不能用有線的 CSMA/CD,改用碰撞「避免」:

  1. CS 載波感測:先聽,別人在傳就等
  2. CA 碰撞避免:頻道空了還要隨機退避(backoff)才發,避免大家齊發 → 效率低但必要
  3. ACK 確認:收到完好訊框立即回 ACK;沒收到就重傳 → 802.11 是少見的「Layer 2 可靠協定」(因無線錯誤率高,值得)
隱藏節點問題 → RTS/CTS

A 和 C 都連 AP(B)但彼此聽不到,CSMA「先聽」失效 → 在 B 處相撞。解法 RTS/CTS(選用):A 先發 RTS 請求,AP 廣播 CTS(大家都聽得到,包括隱藏的 C)宣告禁聲 → A 獨佔時間窗。代價是多一輪握手開銷,故只對大訊框啟用。

速度三引擎:更寬頻道(20→40→80→160 MHz,但頻道數同步減半)、MIMO 多空間流、更高階調變。速度 = 頻寬 × 流數 × 調變效率。

🔑 MIMO 的漂亮逆轉

MIMO 在同頻道同時送多條空間流(N 流 = N 倍速)。靠什麼分開?靠各天線經不同多路徑到達的時間/相位差解聯立方程式。多路徑這個 WLAN 頭號敵人,被 MIMO 變成提速的工具! 好處:更快 + 更遠(波束成形)。

波束成形(Beamforming):多天線以相位差發射,在目標方向同相增強、其他方向抵消——軟體聚焦,可毫秒切換指向。MU-MIMO:用空間分離讓 AP 同頻道同時服務多台設備(空間成為頻率、時間外的第三復用維度)。

⚠ 標稱速率 vs 實際吞吐(必考)

三層折扣:① 協定開銷(CSMA/CA 等待)砍 30–50%;② 總體吞吐由 BSS 內所有人均分(個人 ≪ 總體);③ 距離遠 → SNR 降 → 自動降速。所以「AX5400」隔兩道牆跑 80 Mbps 完全正常。向下相容讓舊設備能連,但會被慢設備拖累全場。

p.86–88無線網狀網路(Mesh)

傳統架構每台 AP 都要拉網路線回 DS。Mesh 拿掉 DS:AP(甚至一般設備)之間用無線互相多跳轉發(multi-hop)訊框,只需少數節點接有線。優點:免佈線、自癒(節點掛了自動繞路)、擴大覆蓋。代價:每跳佔一次頻道時間 → 吞吐逐跳折半、延遲增加。標準 802.11s(不成熟),現代家用 Mesh(Deco/Orbi/eero)多用私有協定 + EasyMesh 認證。

p.3–9WLAN 安全威脅

⚠ TJX 資料外洩案(真實案例)

駭客透過佛羅里達分店一個防護薄弱的 AP入侵,再從分店打進總部,下載數百萬筆信用卡號。若當時用 802.11i 安全就不可能發生。教訓:保護你的端點(endpoint)!

差別:War driver 只「看」(合法);drive-by hacker 會「動手」(非法)。

舊方法都不安全(p.281)

關閉 SSID 廣播、只接受預先核可的 EUI-48(MAC)位址——這兩招都不安全!SSID 仍會在其他訊框中出現、MAC 位址可被竊聽後偽造。真正的安全要靠 802.11i 加密。

p.10–11802.11i 安全(WPA2)

802.11i 在無線主機與 AP 之間提供:初始認證、訊息加密(機密性)、認證、訊息完整性,使用強密碼學標準(AES 加密)。對應 CIA:Confidentiality(機密)、Integrity(完整)、Authentication(認證)。

保護範圍的限制(必考)

802.11i 只保護「無線那一段」(host↔AP),不是端到端加密。封包進了有線網路就是明文。

🔑 設定 AP 的選擇順序

✅ 選 802.11i(=WPA2);❌ 不要選 WPA(Wi-Fi 聯盟的早期較弱方案);⛔ 絕對不要選 WEP(802.11 工作小組的最早期標準,已徹底破解)。安全強度:WPA2 > WPA > WEP。

PSK 初始認證:所有 PC 共用同一把預共享金鑰做初始認證。認證後 AP 會給每台主機一把不共享的會話金鑰(unshared session key),各自加密,彼此不知道對方的金鑰。

⚠ PSK 模式的威脅

① 有人可能把 PSK 給未授權者——它「看起來不像密碼」,員工容易隨意分享(人為脆弱);② PSK 由通關密語產生,密語至少要 20 字元;③ WPS(8 位數 PIN 簡化設定)易被破解,應在 AP 上關閉。

802.1X 模式有三角色:supplicant(請求者=客戶端)、authenticator(認證者=交換器/AP)、中央認證伺服器。有線(UTP)上的鏈路是安全的;但無線段需額外保護 → 在 AP 與客戶端間建 SSL/TLS VPN 保護後續 802.1X 訊息交換,最流行的標準是 PEAP(微軟開發)。

p.21–22WPA3 與 SAE

WPA3(Personal 與 Enterprise)用 SAE 取代 PSK 做更好的認證,提升密碼強度(AES-CCMP 128、HMAC-SHA256;Enterprise 達 192 位元),並使用受保護管理訊框(PMF)。

🔑 SAE(Simultaneous Authentication of Equals)

基於密碼的安全金鑰協商方法,密碼與 MAC 位址共同參與金鑰產生。特點:可自然選密碼、抵抗離線字典攻擊、易用、具前向保密(Forward Secrecy)——即使密碼日後外洩,先前攔截的流量仍無法解密。

p.24–33802.11i 擋不住的威脅

802.11i 只防部分威脅,還有兩種要另外處理:

Rogue Access Point(流氓 AP)

員工未經許可私接的 AP(通常為了方便),往往沒設安全,變成繞過企業防護的後門——攻擊者從這個洞長驅直入。

Evil Twin(邪惡雙胞胎)

⚠ 經典中間人攻擊(MITM)

攻擊者的電腦偽裝成合法 AP(同 SSID),分別與「受害客戶端」和「真 AP」各建一條 802.11i 連線。Evil Twin 解密→讀取→再加密轉發,受害者毫無察覺。因為金鑰是各段分別協商的,ET 握有兩段的金鑰。

✓ 用 VPN 打敗 Evil Twin

客戶端先用 VPN 金鑰加密,再用與 ET 共享的金鑰加密。ET 拆掉外層後,裡面還是 VPN 加密的密文,讀不到原文。VPN 金鑰從未在無線上傳輸,ET 無法攔截 → 端到端保護擊敗中間人。

p.35–42AP 部署與遠端管理

部署規劃:在藍圖上畫約 10 公尺重疊的圓達到良好覆蓋並最小化 AP 間干擾;考慮厚牆、檔案櫃;多樓層要做三維考量。安裝後做場勘(site survey)量測訊號品質、持續調整(環境一直在變)。

遠端管理:AP 是「受管裝置」,把資料送到管理主控台,管理員可下指令。期望功能:故障即時通知、持續品質監控、遠端功率調整、推送軟體更新、自動偵測 Rogue/Evil Twin/設定錯誤的 AP——盡量自動化(人工管理大量 AP 太貴)。

p.63–68NFC · RFID · Wi-Fi Direct

NFC(近場通訊):極短距離(≤ 4 cm)、低速(≤ 424 kbps),故極省電,用 13.56 MHz 專用頻段。應用:公車票、解鎖車門、門禁、電子名片、零售支付(含集點優惠)。

被動 RFID 標籤:目標是取代條碼。標籤無電源,被讀取器掃描時用掃描的能量產生回應 → 便宜,但只能送少量資訊、無法加密。(成本約 4 美分)

Wi-Fi Direct:兩台 802.11 無線主機直接通訊,不需 AP。

TCP/IP 互連網路 I

TCP/IP 標準 · IPv4 位址與遮罩 · 路由程序 · ARP · IPv4/IPv6 封包 · TCP/UDP

p.4–7TCP/IP 標準總覽

p.14–21網路與子網遮罩(Masks)

🔑 為什麼需要遮罩?

看一個 IPv4 位址無法得知網路/子網/主機各佔幾位元(只知總和 32)。遮罩就是解答:一連串開頭的 1 接著一連串結尾的 0,共 32 位。如同噴漆模板——讓部分通過、擋住其餘。

  • 16 個 1 + 16 個 0 = 255.255.0.0 = 前綴 /16
  • 24 個 1 + 8 個 0 = 255.255.255.0 = /24

遮罩運算 = 位元 AND:遮罩位 = 1 時結果取 IP 位元;遮罩位 = 0 時結果一律 0。

目的 IP : 128.171. 17. 13
遮罩/16 : 255.255.  0.  0
結果    : 128.171.  0.  0   ← 取出網路部分

每個路由器有一張路由表,每列代表一段 IP 範圍的路由規則。欄位:列號、Destination(目的網路)、Mask、Metric(路由品質/成本)、Interface(送出埠)、Next-Hop Router(下一跳)。

🔑 封包到達時的三步驟

Step 1 找出所有匹配列:對每一列(從第一列到最後一列全看)套用遮罩,若結果 = Destination 欄 → 該列匹配。
Step 2 找最佳匹配列:基本規則選最長匹配(遮罩開頭 1 最多 = 前綴最長);若平手再比 Metric(成本則選最低、速度則選最高)。
Step 3 送出:依最佳列的 Interface 與 Next-Hop 送出;若 Next-Hop = Local 表示目的主機就在那個介面子網上,直接以訊框送達。

p.60–64位址解析協定(ARP)

🔑 ARP 解決的問題

路由器要把封包送給下一跳,知道對方的IP 位址(來自路由表),但不知道對方的資料鏈結層(MAC)位址——而封包必須裝進該子網適用的訊框。ARP 就是「用 IP 問出 MAC」的協定。

  1. 廣播 ARP 請求:「IP 10.19.8.17 你的 EUI-48 是多少?」
  2. 只有該 IP 的主機回應:「我的 EUI-48 是 A7-23-DA-95-7C-99」
  3. 路由器把 IP↔MAC 配對存進 ARP cache,之後不用再為同一 IP 跑 ARP

p.5–12IPv4 子網切割(Subnetting)

公司拿到網路部分後,把剩餘位元分成子網部分與主機部分。此消彼長:子網部分越大 → 子網越多,但主機部分越小 → 每子網主機越少。

🔑 2ᴺ − 2 公式

N 位元可表示 2ᴺ − 2 個子網或每子網主機數。減 2 是因為全 0(初始未指派)與全 1(廣播)不能用。例:8 位 → 2⁸−2 = 254;12 位 → 4094;16 位 → 65,534。

計算範例(總長固定 32)

網路部分 16 → 剩 16 給公司分配
選 子網/主機 = 8/8 → 子網 2⁸−2=254,每子網主機 2⁸−2=254
選 子網/主機 = 6/10 → 子網 2⁶−2=62,每子網主機 2¹⁰−2=1022

練習:網路 20 位、子網 6 位 → 主機 = 32−20−6 = 6 位 → 主機數 2⁶−2 = 62;子網數 2⁶−2 = 62。

p.14–26網路位址轉換(NAT)

私有 IP 位址(只能內部用,絕不上 Internet):10.x.x.x、172.16–31.x.x、192.168.x.x(最常見)。

🔑 NAT 運作

NAT 防火牆把外送封包的真實內部來源 IP + 埠號記進轉換表,替換成假的外部 IP + 埠號送出;回程封包反向還原。對內外主機透明(transparent)——兩端都不知道 NAT 存在,無須修改主機。

兩大目的:① 擴充可用 IP——每個 IP 約有 4000 個臨時埠,一個 IP 可支撐約 4000 條外部連線(248 個 IP × 4000 ≈ 百萬連線);② 安全——外部嗅探器只能讀到假的外部 IP,無法用來攻擊內部主機。

⚠ NAT 的協定問題

IPsec、VoIP 等需要知道對方真實 IP/埠的應用,難以穿越 NAT 防火牆(NAT traversal),需要特別處理。

p.28–39網域名稱系統(DNS)

網域(domain)= 某組織控制下的一組資源。DNS 是 Internet 的通用命名系統,把名稱解析成 IP。

🔑 DNS 查詢流程(必考)

① 主機要 dakine.pukanui.com 的 IP,問自己的本地 DNS 伺服器;② 本地 DNS 若沒有,轉送請求給 pukanui.com 的權威 DNS 伺服器;③ 權威伺服器把回應送回本地 DNS 伺服器(不是直接給客戶端);④ 本地 DNS 伺服器再把答案送回原始主機。重點:永遠由本地 DNS 伺服器回覆客戶端。

DNS 階層:根(所有網域,有 13 個根伺服器)→ 頂級域名(通用 TLD 如 .com/.edu/.gov 表組織類型;國家 TLD 如 .au/.tw)→ 第二級域名(microsoft.com,競爭激烈)→ 子網域 → 主機。2012 年起任何人可申請管理通用 TLD。

p.69–73網際控制訊息協定(ICMP)

p.21–32行動電話服務

p.47–51PSDN 與 Carrier Ethernet

  • E-LAN:把站點乙太 LAN 延伸覆蓋多個站點

p.52–59多協定標籤交換(MPLS)

🔑 MPLS 為什麼便宜

傳統路由每個封包都跑 Ch8 的三步驟(即使目的相同)→ 耗大量處理力 → 昂貴。MPLS:事先為一段 IP 範圍找好最佳路徑並給一個標籤號(label);每個封包帶標籤,路由器只做快速查表(幾乎不耗處理力)→ 比傳統路由便宜很多。

標籤位於訊框標頭與 IP 封包標頭之間。Label-Switching Router(LSR)依標籤把封包送出對應介面(如 Label 123 → Interface 1)。

p.4–5應用架構與用戶端演進

應用層功能如何分散在電腦間提供服務,三種架構:

  • 獨立運作(Stand-alone)
  • 主從架構(Client/Server)
  • 點對點(P2P)

用戶端演進:啞終端(無桌面處理力)→ Client/Server(部分處理力)→ P2P(大量桌面處理力)。處理力下放是 P2P 興起的條件。

p.6–11應用安全:XSS 與 SQL Injection

⚠ 跨站腳本攻擊(XSS)

使用者輸入「Pat」,網站顯示「Hello Pat」。攻擊者改輸入 <script>alert(document.cookie)</script> 來竊取 cookie。網頁載入時把使用者的「name」放進 HTML,瀏覽器執行了那段腳本。XSS 也可藏在 email 的超連結中(配合 spear phishing)。

⚠ SQL 注入攻擊

原始查詢 SELECT * FROM users WHERE name='[輸入]' AND pw='[輸入]'。攻擊者帳密都輸入 ' OR '1'='1 → 條件恆真 → 繞過驗證登入。根因:把使用者輸入直接拼進 SQL 字串。

應用被入侵的後果

若可被入侵的程式擁有 root 權限,攻擊者可為所欲為——「effectively owns the box(等於擁有整台機器)」。

p.12–28電子郵件標準

🔑 兩套標準(本章核心主題)

① 傳遞標準(Delivery):SMTP(寄送,近乎即時)、POP / IMAP(收件者稍後下載);② 文件格式標準(Document):RFC 822/2822(純文字)、HTML(格式)、ASCII。送得到沒用,收件者還要看得懂。

流程:寄件 client →(SMTP)→ 寄件方 mail server →(SMTP)→ 收件方 mail server →(POP/IMAP)→ 收件 client。

SMTP 對話(命令-回應)

S: HELO voyager...          R: 250 mail.panko.com
S: MAIL FROM: david@...     R: 250 OK
S: RCPT TO: ray@panko.com   R: 250 OK
S: RCPT TO: lee@panko.com   R: 550 No such user here

Web-based E-mail:用 HTTP 取代 SMTP/POP/IMAP,訊息文件是 HTML 網頁。(寄件端 client↔mail server 之間仍可能用 SMTP/POP,但使用者←→自己的 mail server 全走 HTTP)。

p.29–35電子郵件安全

病毒、蠕蟲、木馬常透過附件傳播;防毒軟體幾乎普及但只能部分有效(「antivirus」其實掃描多種惡意軟體)。在兩個以上位置過濾(如 mail server + 收件 client)可達縱深防禦(defense in depth)。

加密的限制

即使寄件者用 SSL/TLS,也只保護到自己 mail host 的那一段(link encryption)。mail server 會解密處理再(可能)重新加密。端到端加密(如 PGP)可行但罕見,因為它會妨礙防火牆過濾訊息。

Open Mail Relay(開放轉送):允許任何人透過它寄信的 SMTP 伺服器,常被濫發垃圾郵件,多被關閉或列入黑名單。應設定只轉送:本地→本地、本地→外部、外部→本地、以及已認證授權的客戶端。

p.37–45VoIP(IP 語音)

在 IP 網路上傳語音電話,省錢:不必維護獨立的語音+資料網路,且語音數位化時被壓縮。

🔑 Signaling vs Transport(電信核心概念)

Signaling(信令):建立連線、結束、計費等;Transport(傳輸):語音/影像的實際傳送。SIP(會話啟始協定)是信令協定,負責所有信令任務(開、關、計費)。傳輸則直接在通話雙方之間進行。

傳輸用 UDP(沒時間等遺失封包重傳,輕量適合即時),搭配 RTP 標頭補足 UDP 兩個缺點:加序號、加時間戳(決定 codec 位元組何時播放)。Media Gateway 連接 IP 網路與 PSTN,處理傳輸與信令轉換。

Codec:把類比語音轉成數位訊號。G.711 = 64 kbps(初始電話標準),G.729AB = 8 kbps,G.723 ≈ 5–6 kbps。壓縮越多 → 音質越差。

p.47–55HTTP 與 HTML

又是「兩套標準」:HTTP = 傳輸標準(檔案傳輸)、HTML = 文件格式標準。瀏覽器送 HTTP Request,伺服器回 HTTP Response + HTML 文件。

🔑 下載複雜網頁(必考)

網頁有文字 + 2 張圖:HTML 檔含文字與圖的標籤(tag)。瀏覽器依標籤分別下載各圖檔再渲染。每個物件一個檔案。例:3 張圖 + 1 首背景音樂 + 1 個 HTML = 5 個檔案。

HTTP 請求:GET /panko/home.htm HTTP/1.1
          Host: voyager.shidler.hawaii.edu
HTTP 回應:HTTP/1.1 200 OK ...
          Content-type: text/plain
          (檔案內容)

CRLF = CR(0x0D)+ LF(0x0A),標記文字檔換行。Windows 用 CRLF、Unix 用 LF、舊 Mac 用 CR。

p.57–63主從架構與 P2P

Client/ServerP2P
處理工作伺服器做繁重處理客戶端互相提供服務
優點中央控制使用者自由、省客戶端容量

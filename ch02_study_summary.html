🌐 網際網路的起源 — ARPANET & NWG

ARPANET 由四個大學節點組成，透過 50 kbps 線路與 IMP（Interface Message Processor）連接。

NWG（Network Working Group）制定規範，稱為 RFC（Requests For Comments），採平等主義（egalitarian）、rough consensus 原則，後演變為 IETF。

⚖️ Standard = Protocol

網路標準（協定）規範訊息的語意（semantics）、語法（syntax）、順序（order）、可靠性（reliability）與連線導向性（connection orientation）。

標準促進互通性（Interoperability）→ 競爭 → 降低價格 → 提升功能。

📨 訊息排序 — HTTP Request/Response Cycle

HTTP 伺服器在收到 HTTP Request 之前不得傳送 Response。

TCP 三向交握開啟連線（SYN → SYN/ACK → ACK），四向交握關閉（FIN/ACK 雙向），遺失時逾時重傳。

🧱 訊息語法（Message Syntax）

訊息由 Header + Data Field（+ Trailer） 組成。Header/Trailer 含多個 Fields。

Ethernet 用固定 octet 長度分隔欄位；HTTP 用 CRLF 分隔欄位、空白行分隔 Header 與 Body。

octet = 8 bits = 1 byte。網路協定以 octet 計算欄位長度。

💡 編碼（Encoding）

ASCII 7-bit 編碼英數字與控制字元（CR = 0x0D, LF = 0x0A），第 8 bit 閒置。

二進位 n bits → 2ⁿ 種狀態。IPv4 = 32 bits → 約 43 億個地址。

語音 麥克風 → 類比訊號（analog） → Codec → 數位訊號 → 傳輸電路。

📦 封裝（Encapsulation）

來源主機從上到下逐層包裹：HTTP Message → TCP Segment → IP Packet → Ethernet Frame → 實體訊號。目的主機反向解封裝（Decapsulation）。

🤝 TCP 三向交握（Three-Way Open）

① A → B：SYN（同步請求連線）
② B → A：SYN/ACK（確認 A 的 SYN，同時發自己的 SYN）
③ A → B：ACK（確認 B 的 SYN）→ 連線建立完成

👋 TCP 四向關閉（Four-Way Close）

A 送 FIN → B 回 ACK → B 可繼續送 Data → B 送 FIN → A 回 ACK → 連線關閉。

🔢 Port Numbers

Well-Known Port（0–1023） 伺服器使用：HTTP=80、HTTPS=443、SMTP=25、FTP=20/21

Ephemeral Port（1024–4999） 客戶端每次連線隨機指派，用完即棄

Socket = IP Address : Port Number（例如 60.171.18.22:2707）

🖧 Ethernet Frame 欄位

Dest EUI-48 (48 bits) → Src EUI-48 (48 bits) → Tag Protocol ID（選，2B）→ Tag Control（選，2B）→ Length (2B) → LLC Subheader (8B) → Packet（Data Field） → PAD → FCS / Frame Check Sequence (4B)

FCS 位於 Trailer，用於錯誤偵測。EUI-48 舊稱 MAC Address。

🌐 IPv4 Packet 重點欄位

Version(4b) · Header Length(4b) · Diff-Serv(8b) · Total Length(16b) · Identification(16b) · Flags(3b) · Fragment Offset(13b) · TTL(8b) · Protocol(8b) · Header Checksum(16b) · Src IP(32b) · Dest IP(32b)

Unreliable Header Checksum 偵測到錯誤 → 封包直接丟棄，不重傳。

📡 TCP Segment Flag Fields

Flag Fields 共 6 個，各佔 1 bit，值 1 = set，值 0 = not set。
包含：SYN ACK FIN 以及其他三個控制旗標。

📊 UDP Datagram 欄位

Source Port(16b) · Dest Port(16b) · UDP Length(16b) · UDP Checksum(16b) · Data Field

比 TCP 簡單許多，無連線建立、無 ACK、無重傳。

🔤 HTTP Request 格式

GET /path HTTP/1.1[CRLF]
Host: hostname[CRLF]
Connection: Keep-Alive[CRLF]
[CRLF]（空白行，結束 Header）

HTTP 用 CRLF（CR=0x0D, LF=0x0A）分隔欄位，空白行後為 Body。

📥 HTTP Response 格式

HTTP/1.1 200 OK[CRLF]
Date: ...[CRLF]
Content-type: text/plain[CRLF]
[CRLF]
（File content）

核心術語表
RFC
Requests For Comments，IETF 網路標準文件，平等主義 + rough consensus 制定
IETF
Internet Engineering Task Force，由 NWG 演變而來的網路標準組織
Interoperability
不同廠商設備依相同協定能互相運作的能力（互通性）
Protocol / Standard
規範訊息語意、語法、順序、可靠性與連線性的規則集合
Segment
TCP 的訊息單位；應用層訊息被分段後的片段
Datagram
UDP 的訊息單位
Packet
IP 層的訊息單位，含 IP Header + TCP Segment（Data Field）
Frame
Data Link 層（Ethernet）的訊息單位，含 Eth Header + Packet + FCS Trailer
Encapsulation
將一個訊息放入另一個訊息的 Data Field 中（逐層包裹）
SYN / ACK / FIN
TCP Flag Fields（各 1 bit）：SYN=同步、ACK=確認、FIN=結束連線
Socket
IP Address : Port Number 的組合，唯一識別一個通訊端點
Well-Known Port
0–1023，伺服器使用。HTTP=80、SMTP=25、FTP=20/21
Ephemeral Port
1024–4999，客戶端每次連線隨機指派（短暫的）
EUI-48 / MAC Address
48-bit Data Link 層位址，用於 Ethernet Frame 識別設備
Octet
8 bits = 1 byte；網路協定常以 octet 計算欄位長度
CRLF
Carriage Return (0x0D) + Line Feed (0x0A)，HTTP 欄位分隔符
Best-effort (IP)
IP 不保證封包送達；錯誤封包直接丟棄，不重傳
Connection-oriented
有明確開啟/關閉連線（TCP）；Connectionless 則無（HTTP、UDP、IP）
FCS
Frame Check Sequence，Ethernet Trailer 的錯誤偵測欄位（4 octets）
Codec
類比電訊號 ↔ 數位訊號的轉換晶片（Coder/Decoder）
ASCII
7-bit 文字編碼；英數字、標點、控制字元（共 128 種）
Header Checksum
IPv4 封包中用於偵測 Header 錯誤的欄位；錯誤則丟棄封包
LLC Subheader
Ethernet Frame Data Field 開頭的 8 octet 子標頭（Logical Link Control）
IMP
Interface Message Processor，ARPANET 的路由節點設備
封裝流程（Source Host 下行）

應用層訊息從 L5 開始，每往下一層被包入更大的訊息單位，最終轉為實體訊號送出。

L5 Application HTTP Message
↓ 封裝入 TCP Segment（Data Field）
L4 Transport HTTP Message TCP Hdr
↓ 封裝入 IP Packet（Data Field）
L3 Internet HTTP Message TCP Hdr IP Hdr
↓ 封裝入 Ethernet Frame（Data Field）
L2 Data Link Eth Hdr IP Hdr TCP Hdr HTTP Msg FCS Trlr
↓ 轉換為實體訊號（Physical Layer）
L1 Physical 0101101010111001... （電/光/無線訊號）
🔄 目的主機反向解封裝

L1 收訊號 → L2 讀 Eth Header/FCS，取出 Packet → L3 讀 IP Header，取出 Segment → L4 讀 TCP Header，取出 HTTP Message → L5 應用程式處理。

🔁 二進位轉換快速複習

位元數 n → 可表示 2ⁿ 種值。
8 bits = 256 種 · 16 bits = 65,536 種 · 32 bits = 4,294,967,296 種（IPv4 地址數量）

二進位 → 十進位：各位元值 × 對應位值加總（1·128 + 0·64 + 1·32 ...）
十進位 → 二進位：從最大位值開始貪心減法，直到餘數為 0。

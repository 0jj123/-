QoS 指標 §4.2–4.5
⚡
Speed（速度）
以 bps 衡量的傳輸速率。分為 Rated Speed（廠商聲稱或標準定義）與 Throughput（實際使用者可得速率）。
🎯
Errors（錯誤率）
不正確的 bits 或 packets 佔總數的百分比。
🟢
Availability（可用性）
網路對使用者可用的時間百分比。
PSTN ≈ 99.999%（Five Nines）。
⏱️
Latency（延遲）
封包傳遞的時間延遲，以 ms 衡量。
1 ms = 0.001 s；1 µs = 10⁻⁶ s。
📉
Jitter（抖動）
延遲的變異性（variability in latency）。對 VoIP、視訊、互動應用影響巨大；對 e-mail 等傳統應用影響不大。
📋
SLA
服務等級協議，以最壞情形規格（worst-case）表示，例如「不低於 100 Mbps，99.5% 的時間」。企業用戶比家用用戶更需要高保證。
Throughput 層次關係
Rated Speed
最外圈 — 技術標準或廠商宣稱值（最大值）
Aggregate Throughput
網路區段所有使用者共享的總實際吞吐量
Individual Throughput
最內圈 — 單一使用者從 Aggregate 中分得的份額
📝 計算範例（Speed Knowledge Check）
AP 額定 300 Mbps，Throughput ≈ 50% → Aggregate = 150 Mbps
5 人同時使用（含自己）→ Individual = 150 ÷ 5 = 30 Mbps
網路設計 §4.6–4.8
Two-Site 流量分析
原則
大多數傳輸線為對稱（symmetric）速率。若兩方向需求不同，以較高速率決定線路規格。
例：A→B 需 100 Mbps、B→A 需 1 Gbps → 選 1 Gbps 線路。
Three-Site 流量分析（串聯 Q→R→S）
流量對需求頻寬影響的 Link
Q ↔ R45 MbpsLink Q-R 直接承載
R ↔ S2 GbpsLink R-S 直接承載
Q ↔ S300 Mbps必須穿越 Link Q-R 與 Link R-S
Link 頻寬計算
Link Q-R 所需頻寬45 + 300 = 345 Mbps
Link R-S 所需頻寬2000 + 300 = 2300 Mbps（2.3 Gbps）
Three-Site with Redundancy（全互連）
優點
每對站點有直連線路，各條 Link 只承載該對的流量 → 不需疊加轉送流量，但需要更多線路。

Line 1 (Q-S): 300 Mbps / Line 2 (Q-R): 45 Mbps / Line 3 (R-S): 2 Gbps
Momentary Traffic Peaks 應對策略
方法說明評價
Overprovisioning 提供遠超正常需求的容量 簡單有效，但浪費頻寬成本
Priority 延遲敏感流量（VoIP）優先；容忍延遲流量（e-mail）等待 更有效率，但需大量人工設定
Reserved Capacity 為特定流量保留固定容量，其他流量競用剩餘 保證關鍵流量，其他流量可能受限

六大安全規劃原則
  • Risk Analysis（風險分析）：目標不是消除所有風險，而是將風險降至經濟合理的水準。需比較對策效益與對策成本（Annual Value = 無對策支出 − 有對策支出 − 對策年費）。
  • Comprehensive Security（全面安全）：封閉所有攻擊管道（Avenues of Attack），一個開口就足以讓攻擊者得逞。
  • Defense in Depth（縱深防禦）：多道對策串聯，任一對策成功即可阻擋攻擊。邊境防火牆失守 → Host 防火牆接手 → 漏洞修補 → 加密資料。
  • Weakest Link Analysis（最弱環節分析）：單一對策由多元件組成，全部元件必須正常才能有效。攻擊者只需找到最弱環節即可突破。
  • Single Points of Takeover（保護單一接管點）：集中控制伺服器（如防火牆控制服務器）若被攻陷，可控制所有下游設備 → 應特別加強保護。
  • Least Permissions（最小權限）：每位已驗證使用者只授予完成工作所需的最低權限，防止越權操作。Authentication（身份驗證）≠ Authorization（授權）。

集中管理 §4.18–4.22
Ping
用途
測試主機或路由器是否可達（reachable）。傳送 ICMP Echo Request，等待 Echo Reply。
範例：ping -n 20 -4 www.ibm.com　→ 顯示 RTT 最小/最大/平均值。
SNMP (Simple Network Management Protocol)
Manager
網路管理軟體，發送 Get/Set 等 SNMP 指令，接收 Response 與 Trap。
Agent
裝置上的管理代理程式，存取本機 MIB，回覆指令，並在異常時主動送出 Trap（告警）。
MIB
Management Information Base — 儲存裝置管理資訊的資料庫，位於被管理裝置端（非 Manager 端）。
Trap
Agent 主動向 Manager 發出的告警訊息（非 polling）。
Network Visualization
分析 MIB 資料，圖形化呈現網路拓樸、進行故障排除。SNMP 互動已標準化，但視覺化程式功能不標準化（保留開發彈性）。

Centralized Firewall Management
架構
Firewall Policy Server 持有統一 Policy Database → 自動轉換為各防火牆專屬規則（Firewall-Specific Rules）下發。

好處：降低人工成本、加快部署速度，確保政策一致性。
風險：Policy Server 本身成為 Single Point of Takeover，需特別保護。

Flashcard Hover to flip
Q
Jitter 的定義是什麼？
A
Jitter 是延遲的變異性（variability in latency）。封包間隔不一致，使 VoIP/視訊呈現「抖動」感。
Q
SLA 的保證是以什麼形式表達？
A
最壞情形規格（Worst-case）：如最低 100 Mbps 的 99.5% 時間保證。100% 合規成本過高，不實際。
Q
Three-Site 串聯中，Link Q-R 需承載哪些流量？
A
① Q↔R 直接流量 + ② Q↔S 轉送流量。兩者相加為 Link Q-R 所需頻寬（例：45 + 300 = 345 Mbps）。
Q
Defense in Depth vs Weakest Link 的成功條件？
A
• Defense in Depth：任一對策成功即阻擋攻擊
• Weakest Link：所有元件必須正常，攻擊者只需突破最弱點
Q
SNMP 中 MIB 儲存在哪裡？
A
MIB 儲存在被管理裝置（Agent 端），不是在 Manager 端。Manager 透過 Get/Set 指令讀寫 MIB。
Q
Overprovisioning 的缺點是什麼？
A
浪費傳輸線路容量（wasteful of transmission line capacity），需付出多餘的頻寬費用。
Q
最小權限原則（Least Permissions）的核心概念？
A
每位已驗證使用者只授予完成工作所需的最低權限，防止越權操作危害安全。
Q
SDN 使用什麼協定控制轉送裝置？
A
OpenFlow。SDN Controller 透過 OpenFlow 指令集中控制 SDN Switch / Router / Access Point 的轉送行為。
Q
Risk Analysis：何時應採用對策？
A
當對策的年節省金額（Annual Value）> 0 時採用。
計算：無對策年損 − 有對策年損 − 對策年費 > 0。
Q
Comprehensive Security 的意義？
A
封閉所有攻擊管道（All Avenues of Attack）。攻擊者只需找到一個未防禦的入口即可入侵。
小測驗 10 題
得分：0 / 0
Q1Wi-Fi AP 額定 300 Mbps，Throughput 約 50%，目前有 5 人同時傳輸。你的個人吞吐量約為？
300×50%=150 Mbps (Aggregate)，5人平分 = 30 Mbps。
Q2下列哪項是 Jitter 的定義？
Jitter = variability in latency（延遲的變異性）。
Q3Three-Site 串聯（Q-R-S）中，Q↔S 流量需求 300 Mbps，Q↔R 直接需求 45 Mbps。Link Q-R 需要多少頻寬？
Link Q-R 需承載 Q↔R(45) + Q↔S(300) = 345 Mbps。
Q4SLA 通常以哪種方式表達保證？
SLA 以 Worst-case specification 表達，例如「99.5% 的時間不低於 100 Mbps」。
Q5下列哪個說法正確描述 Defense in Depth？
Defense in Depth：多對策串聯，任一成功即阻擋（選項D是 Weakest Link 的描述）。
Q6SNMP 中，MIB（Management Information Base）位於？
MIB 位於被管理裝置端，由 Agent 維護，Manager 透過指令存取。
Q7Risk Analysis 中，對策年節省值（Annual Value）如何計算？
Annual Value = (無對策Net outlay) − (有對策Net outlay) = $200K − $120K = $80K（對策A範例）。
Q8SDN 使用哪種標準協定從 Controller 控制轉送裝置？
SDN Controller 透過 OpenFlow 指令集中控制 SDN Switch / Router / AP。
Q9Protecting Single Points of Takeover 的安全意義是？
集中控制點（如 Firewall Control Server）是 Single Point of Takeover，被攻陷即影響所有下游，需重點保護。
Q10以下哪項不屬於 SNMP Agent 的功能？
網路視覺化（Network Visualization）是獨立程式，分析 MIB 資料後呈現；不是 Agent 的功能。
